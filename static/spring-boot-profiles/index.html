在 Spring Boot 開發時，一般都會分為開發(dev)、測試(test)與生產(prod)環境。通過一份 profile 就可以對各個環境下的配置進行統一的管理。

  • application.yml - 主配置文件
  • application-{name}.yml - 環境配置(掛件)
  • spring.profiles.active:{name} - 激活環境

舉例，現在在 application-dev.yml 中指定了端口號 8080，而在 application-prod.yml 中指定了端口號 80。

- application-dev.yml -
server:
  port: 8088
  
- application-prod.yml -
server:
  port: 80

若在 application.yml 中指定環境：

spring:
  profiles:
    active: dev

那麼現在通過 localhost:8080 就可以訪問到系統。若將其改為 prod：

spring:
  profiles:
    active: prod

就可以通過 localhost 直接訪問 (因為80是 Tomcat 默認端口)。
